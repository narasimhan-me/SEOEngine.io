## 1. Overview

EngineO.ai is an AI-first Discovery Engine Optimization (DEO) platform that combines:

- SEO automation
- AEO (Answer Engine Optimization)
- PEO (Product Engine Optimization)
- VEO (Voice Engine Optimization)
- Entity intelligence
- Answer-ready content
- Multi-engine visibility
- Shopify + web integrations
- Bulk content/product optimization
- Automated DEO playbooks

This stack positions EngineO.ai closer to tools like Surfer, Clearscope, Frase, Ahrefs, DeepCrawl, Perplexity for Business, and AI content orchestration tools ‚Äî but with automation as a core advantage.

The pricing strategy must reflect:

- High-value automation
- AI token variable costs
- Multi-project agency usage
- eCommerce scaling (product-heavy stores)
- AI assistant + search visibility monitoring
- Professional services potential
- Long-term enterprise workflow integrations

This document defines Launch Pricing, Scale Pricing, and Enterprise Pricing, including token strategy, feature gating, and upgrade paths.

---

## 2. Pricing Goals

The pricing model must:

1. **Maximize early adoption**
   - Low-friction entry ‚Üí rapid user acquisition.

2. **Capture value over time**
   - DEO is radically more valuable than "AI SEO tools".
   - As automation + intelligence scale, prices should rise accordingly.

3. **Encourage expansion (land & expand)**
   - Multi-project teams and agencies should grow naturally into higher tiers.

4. **Monetize automation + compute**
   - DEO audits and bulk generation consume compute + AI tokens. Pricing must scale with usage.

5. **Maintain competitive clarity**
   - Offer simple, predictable tiers while supporting advanced capabilities.

---

## 3. Pricing Philosophy

EngineO.ai uses a hybrid value model:

- Subscription-based tiers
- AI token allotments per tier
- Feature gating
- Project and product/page limits
- Automation depth
- Reporting & team access

This aligns with how modern AI SaaS tools scale (e.g., Jasper, Frase, Surfer) while also matching eCommerce/agency usage patterns.
